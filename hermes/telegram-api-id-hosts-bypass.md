# Как получить api_id на my.telegram.org из России без VPN

**Автор:** Маша Черноярова ([@monrech](https://habr.com/ru/users/monrech/))
**Дата:** 2026-06-15 (статья от 22 часов назад)
**Источник:** https://habr.com/ru/articles/1047402/

**Рейтинг:** +15, 17K просмотров

## Главная идея

my.telegram.org заблокирован в России **на уровне DNS, а не DPI**. Поэтому можно обойти блокировку без VPN — прописать IP-адрес в `/etc/hosts`.

Проблема с VPN: сайт открывается, но при создании приложения Telegram-антифрод проверяет соответствие страны номера телефона и IP. Российский номер + голландский IP = `ERROR`.

## Решение

```
149.154.167.220 my.telegram.org
```

Добавить в `/etc/hosts` (Linux/Mac) или `C:\Windows\System32\drivers\etc\hosts` (Windows) **с выключенным VPN**, сбросить DNS-кэш, перезапустить браузер, создать приложение, **удалить строку из hosts**.

## Применимость к Smart Agent

| Концепция | У них | У нас |
|---|---|---|
| api_id / api_hash | Для Telethon-парсера каналов | Для Telegram-ассистента Smart Agent (user API) |
| Нужен user API, не Bot API | Чтение каналов как подписчик | Полноценный доступ к чатам/каналам пользователя |
| Российский номер | +7 XXX | У пользователя российский номер |
| Сервер в России | Локальная машина | Hermes-сервер с российским IP |

**Вывод:** когда понадобится api_id для Smart Agent-ассистента — способ из статьи сработает с сервера Hermes без VPN.

## Оценка

✅ Практическая ценность: высокая — готовая инструкция на случай необходимости
⚠️ Актуальность: IP `149.154.167.220` может измениться, нужно проверять на момент использования
⚠️ Способ работает пока блокировка только DNS, не DPI
