# You don't need sub-agents

**Автор:** Anton Vinogradov ([@tony__vi](https://dev.to/tony__vi))
**Дата:** 2026-06-16
**Источник:** https://dev.to/tony__vi/you-dont-need-sub-agents-1eh7

---

## Главная идея

Диаграмма с Оркестратором → Researcher, Coder, Tester, Reviewer — это не архитектура. Это org chart. Single loop с хорошими инструментами beats sub-agents.

## Ключевые положения

- **Закон Брукса (1975):** n(n−1)/2 линий коммуникации. Оркестратор субагентов = накладные расходы
- **Контекст не выживает при делегировании.** Субагент возвращает сжатое резюме — рассуждения заперты в изолированном окне
- **Токены:** разветвление на N агентов стоит N+1
- **PostHog:** год продакшена → single loop > sub-agents
- **Cognition (Devin):** «Don't Build Multi-Agents» → «работает только когда пишет один поток»
- **Berkeley MAST:** 41-87% отказов в multi-agent фреймворках. Проблема в coordination, не в модели
- **Правило:** параллелить только независимые задачи. Без оркестратора. Два отдельных процесса.

## Применимость к Hermes

| Концепция | У них | У нас |
|-----------|-------|-------|
| Single loop | Основной паттерн | `main_loop()` — единый цикл |
| Sub-agents | Только для независимых задач | `delegate_task` для параллельного аудита |
| Оркестратор | Не нужен | У нас его и нет — агент сам всё делает |
| Shared context | Файлы/git как костыль | Obsidian vault + SQLite — общая память |
