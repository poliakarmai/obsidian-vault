# Sakana Fugu — оркестрационный слой из нескольких LLM

Источник: пользователь, 22.06.2026
Ссылка: https://sakana.ai/fugu

## Концепт

Fugu — это обученная LLM-координатор, которая:
1. Принимает промпт пользователя
2. **Динамически решает** — ответить самой или передать подзадачи другим моделям
3. Распределяет работу между пулом LLM (включая рекурсивные экземпляры себя)
4. Собирает результаты → единый ответ

Единый OpenAI-совместимый endpoint.

## Что это значит для @Apolaibot

Fugu = оркестратор LLM. Hermes = оркестратор агентов/скиллов. Та же архитектурная идея, на уровень ниже.

**Что можно взять:**
- **Модель-роутер** — Fugu сама решает кому делегировать. У нас роутер скиллов работает по правилам — можно обучить LLM-роутер как Fugu
- **Единый endpoint** — пользователь не знает какая модель отвечает. Так же как тенант не знает какой скилл отработал
- **Рекурсивная оркестрация** — Fugu может вызывать саму себя. У нас delegate_task → саб-агенты

**Разница:** Fugu оркестрирует модели, Hermes оркестрирует инструменты. Но паттерн один и тот же.
