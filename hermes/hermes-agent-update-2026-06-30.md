## AsyncSessionDB (PR #54876)

**Проблема:** синхронные SQLite-запросы в gateway блокировали event loop.
Один цикл обслуживает все сессии + heartbeat + Kanban.
Блокирующий SQL-вызов замораживал ВЕСЬ цикл.

**Решение:** AsyncSessionDB — обёртка SessionDB.
Каждый метод выгружается в отдельный рабочий поток.
Одна дверь — одна модель исполнения.

**Эффект для нас:**
- Меньше зависаний при параллельных сессиях
- Сигналы активности не блокируются
- Kanban-задачи не стопорят чат
