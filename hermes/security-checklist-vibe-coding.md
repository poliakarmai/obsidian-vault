# Чеклист безопасности для вайбкодеров

От разработчика с 20+ лет опыта. Закрыть базу перед релизом.

## 1. Юридическая защита
Как только собираешь данные пользователей → GDPR, CCPA. Политика конфиденциальности обязательна. Знать где хранятся данные.

## 2. Row Level Security (Supabase)
Без RLS любой может открыть DevTools и прочитать всю БД. Auth → Policies. Если политик нет — база открыта.

## 3. Тестируй неуспешные сценарии
- Неправильный пароль 5 раз
- Сброс пароля для несуществующей почты
- Дважды открыть ссылку подтверждения
- Регистрация с существующим email

## 4. Заголовки безопасности
Запрос: «Проверь моё приложение как специалист по безопасности и убедись, что у меня сильные заголовки безопасности и надёжный базовый уровень безопасности».

## 5. OWASP
Запрос: «Проверь моё приложение на соответствие стандартам OWASP и подсвети уязвимости». Ловит SQL-инъекции, XSS, ошибки аутентификации.

## 6. Валидация на сервере
Клиентскую валидацию можно обойти (отключить JS). Все проверки дублировать на сервере.

## 7. Три типичные утечки AI-кода
- Значения из .env попадают во фронтенд
- API возвращает больше данных чем нужно
- Секреты в логах

Запрос: «Проверь моё приложение на утечки credentials или чувствительных данных во фронтенде или API-роутах».

## 8. API-ключи ≠ фронтенд
Ключ в браузере = скомпрометирован. Вынос на сервер или прокси.

## 9. Rate limiting до релиза
Все эндпоинты к платным API под лимиты. Один неконтролируемый поток: $20 → $200 за день.

## 10. CAPTCHA + CORS
Cloudflare Turnstile (бесплатный) на публичные формы. CORS ограничить своим доменом. ~10 минут работы.

## 11. Не раскрывай внутренние ошибки
❌ «SELECT * FROM users failed»
✅ «User not found»
Детали — только в серверные логи.
