# GSC Roadmap — от инструмента к продукту

> Состояние: 25.06.2026. 6 проектов отaudit'овано, 100+ находок в DB, самообучение работает.

## Фаза 1: Упаковка (июль 2026) — 2 недели

**Цель:** GSC можно показать и он не выглядит как «скрипт на коленке».

| # | Задача | Сложность | Результат |
|---|--------|-----------|-----------|
| 1.1 | `gsc scan <project>` — единая CLI-команда | 🟢 низкая | Вместо 3 delegate_task вручную — один вызов |
| 1.2 | 200 seed-паттернов (OWASP Top 10, CWE Top 25) | 🟡 средняя | Первый аудит не слепой — базовые правила есть всегда |
| 1.3 | Веб-дашборд (история, тренды, график) | 🟡 средняя | `gsc dashboard` → localhost:8080 с графиками |
| 1.4 | `gsc init` — авто-установка в проект | 🟢 низкая | `.gsc/` в корне, gitignore, CI-конфиг |
| 1.5 | README + docs сайт (MkDocs) | 🟡 средняя | Можно дать ссылку, выглядит серьёзно |

**Метрика:** посторонний разработчик может запустить GSC по README и получить первый аудит за 5 минут.

## Фаза 2: Интеграция (август 2026) — 3 недели

**Цель:** GSC работает без участия человека. CI → аудит → отчёт → PR.

| # | Задача | Сложность | Результат |
|---|--------|-----------|-----------|
| 2.1 | GitHub Action: `poliakarmai/gsc-action@v1` | 🟡 средняя | PR → авто-аудит → комментарий с находками |
| 2.2 | GitLab CI шаблон | 🟡 средняя | То же для GitLab |
| 2.3 | `.gsc/config.yaml` — per-project rules | 🟢 низкая | Игнор-лист, кастомные пороги, свои паттерны |
| 2.4 | JSON/Markdown/SARIF экспорт | 🟢 низкая | Совместимость с GitHub Code Scanning |
| 2.5 | Slack/Telegram нотификации | 🟢 низкая | Алерт при CRITICAL находках |

**Метрика:** можно установить GSC Action на любой GitHub-репо и получать аудит в каждом PR.

## Фаза 3: Масштабирование (сентябрь 2026) — 3 недели

**Цель:** GSC работает с чужими проектами, а не только с нашими.

| # | Задача | Сложность | Результат |
|---|--------|-----------|-----------|
| 3.1 | Поддержка Java, C#, Go, Rust, JS/TS | 🔴 высокая | Не только Python. Сначала Go+TS (макс охват) |
| 3.2 | Поддержка инфра-конфигов (Terraform, K8s, Docker) | 🟡 средняя | DevOps-аудиты — больной рынок |
| 3.3 | Авто-фиксы (AI предлагает diff) | 🔴 высокая | «Нашёл TOCTOU — вот патч». Click to apply |
| 3.4 | Pre-commit hook | 🟢 низкая | Локальный аудит до коммита |

**Метрика:** 3+ языка, 50+ пользователей из внешних проектов.

## Фаза 4: Монетизация (октябрь 2026) — 4 недели

**Цель:** GSC приносит деньги, а не жрёт токены DeepSeek.

| # | Задача | Сложность | Результат |
|---|--------|-----------|-----------|
| 4.1 | SaaS: `gsc.cloud` — веб-интерфейс | 🔴 высокая | Регистрация → подключить репо → аудит |
| 4.2 | Тарифы: Free (1 репо, public only) / Pro ($29/мес) / Team ($99/мес) | 🟡 средняя | Бесплатно для опенсорса, платно для приватных |
| 4.3 | On-premise: self-hosted лицензия | 🟡 средняя | Enterprise с air-gap — $5K/год |
| 4.4 | Stripe/Paddle биллинг | 🟡 средняя | Автоматические платежи |

**Метрика:** 10 платных пользователей, $500 MRR.

---

## Фаза 5: Сеть (ноябрь-декабрь 2026) — 6 недель

**Цель:** GSC учится не только на своих аудитах, но и на чужих.

| # | Задача | Сложность | Результат |
|---|--------|-----------|-----------|
| 5.1 | Федеративное обучение: паттерны из N проектов → общий пул | 🔴 высокая | Чем больше юзеров — тем умнее GSC для всех. Приватность: только агрегированные паттерны, не код. |
| 5.2 | Opt-in анонимизация: «разрешить отправлять анонимные паттерны» | 🟡 средняя | GDPR-friendly, пользователь контролирует |
| 5.3 | Паттерн-маркетплейс: шарить/продавать кастомные правила | 🟡 средняя | «PCI DSS паттерны от Security Corp — $99» |

**Метрика:** 1000+ паттернов в общем пуле, точность >90% за счёт кросс-валидации.

## Риски

| Риск | Вероятность | Смягчение |
|------|------------|-----------|
| LLM hallucination в паттернах | Средняя | False-positive learning + порог эффективности 0.3 |
| Медленно на больших репо | Высокая | Сначала только дифф PR, не весь репо |
| Нишевый рынок (AI audit tools) | Средняя | Фокус на самообучение — никто этого не делает |
| Зависимость от DeepSeek API | Высокая | Локальные модели (Llama-4, Qwen-3) как fallback |
| Конкуренты скопируют | Низкая | Сложность не в коде, а в накопленных паттернах |
