### 2.5. 🔄 Продление подписки — полный lifecycle

**Текущий флоу:** оплата → активация на 30 дней → депровижнинг. Работает но глупый: юзер узнаёт об отключении только когда бот перестаёт отвечать. Нужно:

**Фазы подписки:**

```
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│ SIGNUP   │→  │ ACTIVE   │→  │ GRACE    │→  │ EXPIRED  │→  │ CHURNED  │
│ оплата   │   │ 30 дней  │   │ +3 дня   │   │ offboard │   │ >30 дней │
└──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘
                    ↑______________|              ↑______________|
                    auto-renew                    manual renew
```

**SIGNUP** (день 0):
- Пользователь платит через @miropolbot (Stars) или TON (CryptoBot)
- `stars-activator.py` или `ton-activator.py` → `hermes-tenant onboard`
- БД: `pro_users` → `activated_at=NOW, expires_at=NOW+30d, active=1, renewal_count=0`

**ACTIVE** (день 1-30):
- Бот работает, скиллы доступны
- День 23 (за 7 дней): ⚠️ напоминалка «Подписка истекает через 7 дней»
- День 27 (за 3 дня): 🔔 «Продли за 3 дня — иначе бот отключится»
- День 29 (за 1 день): 🚨 «Последний день! Продли сейчас»

**GRACE** (день 31-33, +3 дня после истечения):
- Бот ВКЛЮЧЁН, но с дисклеймером «просрочено, продли»
- Если продлил → сброс в ACTIVE, `renewal_count++`
- Если не продлил → EXPIRED

**EXPIRED** (день 34):
- `deprovision.py`: `hermes-tenant offboard`, бот отключён
- Уведомление: «Подписка истекла. Продлить за 30 мин через @miropolbot»
- Данные пользователя НЕ удаляются (vault, сессии остаются)

**CHURNED** (день 64, +30 дней после EXPIRED):
- Если не вернулся → архивация данных, освобождение песочницы
- Возможность восстановления через поддержку

**RENEWAL (продление):**
- Пользователь получает напоминалку → нажимает «Продлить» → @miropolbot
- Оплата → `billing.py upgrade <tg_id> <tier>` с параметром `renewal=true`
- `expires_at = MAX(текущий expires_at, NOW) + 30d` — не теряем оставшиеся дни
- `renewal_count++` для аналитики

**Что построить:**

| Компонент | Тип | Где |
|-----------|-----|-----|
| Напоминалки (7д/3д/1д) | cron, ежедневно 09:00 | `subscription-reminder.py` |
| Grace period (+3 дня) | логика в deprovision | `deprovision.py` |
| Кнопка «Продлить» в /start | бот @Morearbot | `morearbot.py` |
| Статистика продлений | аналитика | `billing.py dashboard` |
| Архивация churned (>30д) | cron, еженедельно | `subscription-archiver.py` |

**Схема БД (pro_users):**
```sql
ALTER TABLE pro_users ADD COLUMN renewal_count INTEGER DEFAULT 0;
ALTER TABLE pro_users ADD COLUMN last_renewal_at TEXT;
ALTER TABLE pro_users ADD COLUMN reminders_sent INTEGER DEFAULT 0;  -- битовая маска: 1=7d, 2=3d, 4=1d
ALTER TABLE pro_users ADD COLUMN grace_until TEXT;  -- NULL пока не истекла
ALTER TABLE pro_users ADD COLUMN churned_at TEXT;
```

**Уведомления пользователю (шаблоны):**

*За 7 дней:*
```
⚡ Hermes Pro: подписка истекает через 7 дней.
Продли сейчас — бот работает без перерыва.
👉 @miropolbot → /start
```

*За 3 дня:*
```
⚠️ 3 дня до отключения Hermes Pro!
После истечения бот перестанет отвечать.
Продлить сейчас: @miropolbot
```

*За 1 день:*
```
🚨 Завтра бот отключится!
Последний шанс продлить подписку Hermes Pro.
@miropolbot → 100 ⭐
```

*В grace-периоде (день 31-33):*
```
🕐 Подписка истекла, но бот пока работает.
Продли в течение 3 дней — данные сохранятся.
@miropolbot → /start
```

*После EXPIRED:*
```
❌ Подписка Hermes Pro отключена.
Бесплатный бот: @Apolaibot
Продлить: @miropolbot (100 ⭐)
```

**Экономика продлений:**
- Цель: 70% retention после первого месяца
- Авто-продление: опция для юзера (вкл/выкл в /settings)
- Скидка за годовую подписку: 10 × месячных (2 месяца бесплатно)
- Реферальная программа: «приведи друга → +7 дней бесплатно»

### 3. 🛡️ Стабильность оркестрации

**Правила для LLM-джоб (уже в конституции XIV-A):**
- Трейдинг-данные → только no_agent скрипт
- Если LLM-джоба даёт цифры — первым шагом RPC/API вызов
- Нет вызова → нет отчёта (не выдумывать)

**Мониторинг здоровья:**
- `pipeline-trace` (каждые 12ч) — gateway + bybit-ws
- `health-dashboard-hourly` — сервисы + диск + память
- `bots_health_check` (ежедневно 18:00) — все боты

**Авто-восстановление:**
- `gateway-patch-watchdog` (каждые 10 мин)
- `rate-guard` (каждые 15 мин)
- `vpn-watch` (каждые 15 мин)
- Systemd: `Restart=on-failure` + `RestartSec=5`

**Изоляция от мусора:**
- `Session Prune (7d)` — ежедневно
- `memory-prune-daily` — ежедневно
- Контекст-бюджет weekly — аудит

### 4. 🤖 Supervised Agent (Джуниор-оркестратор)

**Архитектура:**
```
Море (V4 Pro, default)
  │
  ├── Трейдинг, VPN, Газпром, Критическое
  │
  └── Джуниор (V3, отдельный профиль "assistant")
       │
       ├── Рутинные вопросы тенантов (погода, поиск, tutu)
       ├── Первичная обработка запросов
       ├── Действует по скиллам Море
       └── Эскалирует сложное → Море
```

**Реализация:**
1. Копируем скиллы Море в профиль `assistant`
2. Настраиваем `channel_prompt` с ролью: «ты джуниор, сложное — Море»
3. `max_concurrent_sessions: 3` только для тенантов
4. Ежедневный авто-саммари его сессий → ревью Море
5. Раз в неделю: апдейт скиллов на основе ошибок

**Критерии эскалации (джуниор → Море):**
- Трейдинг (bybit, позиции, ордера)
- VPN-инфраструктура
- Изменение конфигов/скиллов
- Ошибки в ответах (пользователь жалуется)
- Неуверенность (confidence < 0.7)

**Метрики джуниора:**
- % эскалированных запросов (цель: 20-40%)
- % успешных ответов без эскалации
- Среднее время ответа
- Satisfaction тенантов (👍/👎)

### 5. 🧠 Qwen Brain — 2 модели для сложных задач

**Роль:** параллельный «мозговой штурм» с V4 Pro. Две Qwen-модели как специалисты.

**Архитектура:**
```
Задача (сложный анализ, архитектура, стратегия)
  │
  ├── V4 Pro (основной) → финальный ответ
  ├── Qwen-Plus (аналитика) → анализ рынка, числа, логика
  └── Qwen-Max (креатив) → текст, стратегия, питч
       │
       └── cross-model-review → консенсус → пользователь
```

**Какая модель для чего:**
| Модель | Провайдер | Специализация | Когда |
|--------|-----------|---------------|-------|
| **Qwen-Plus** | qwen/qwen-plus | Аналитика, рассчёты, логика | Бюджеты, PnL-расчёты, бизнес-анализ |
| **Qwen-Max** | qwen/qwen-max | Креатив, стратегия, убеждение | Питчи, стратегия, документация |

**Интеграция:**
- `hermes config set auxiliary.qwen_plus` — как fallback/советник
- `cross-model-review` скилл уже есть — добавить Qwen как второе мнение
- Не заменяет V4 Pro, а дополняет: два мнения → Море выбирает лучшее

**Затраты:**
- Qwen-Plus: ~$0.80/1M input, ~$2.40/1M output
- Qwen-Max: ~$2.40/1M input, ~$9.60/1M output
- Расчётно: +$10-15/мес на Море (эпизодическое использование)
- Для тенантов Pro: +$5/мес если включено

**Конфигурация (custom_providers):**
```yaml
custom_providers:
  - name: qwen-plus
    base_url: https://dashscope-intl.aliyuncs.com/compatible-mode/v1
    api_key_env: DASHSCOPE_API_KEY
  - name: qwen-max
    base_url: https://dashscope-intl.aliyuncs.com/compatible-mode/v1
    api_key_env: DASHSCOPE_API_KEY
```


**Инструменты:**

| Инструмент | Назначение | API |
|-----------|-----------|-----|
| **Qwen-VL** | Vision: анализ изображений, понимание запроса | qwen/qwen-vl-max |

**Сценарии:**
- «Сделай логотип для Hermes» → Banana Pro (gen)
- «Убери фон с этого фото» → Banana Pro (edit) + Qwen-VL (проверка)
- «Сгенерируй обложку для статьи» → FAL.ai (есть) + Qwen-VL (оценка)
- «Что на этой картинке?» → Qwen-VL (анализ)

**Интеграция в Hermes:**
- `image_generate` tool уже есть (FAL.ai)
- Добавить provider для Banana Pro в `custom_providers`
- Qwen-VL уже доступен через OpenRouter как vision-модель
- Скилл: `image-ops` — роутер между FAL, Banana, Qwen-VL

**Затраты:**
- Banana Pro: ~$0.02-0.05 за генерацию (pay-as-you-go)

- Итого: копейки на картинку

### 7. 💼 Продажник — мощный standalone-агент (Qwen + DeepSeek)

**Роль:** автономный sales-помощник с собственным профилем и специализированными скиллами.

**Архитектура:**
```
@salesman (отдельный профиль, Telegram бот)
  │
  ├── DeepSeek V3 (базовая логика, расчёты, CRM)
  ├── Qwen-Max (питчи, креатив, убеждение)
  └── Скиллы:
       ├── business-development (анализ конкурентов, B2B)
       ├── metal-supply-kp (коммерческие предложения)
       ├── marketplace-research (анализ товаров)
       ├── ppt-master (презентации)
       └── email-ops (рассылки, follow-up)
```

**Сценарии:**
- «Составь КП на трубу 904L для Газпрома» → анализирует рынок, делает КП
- «Найди 5 потенциальных клиентов в Екатеринбурге» → OSINT + анализ
- «Сделай презентацию для инвесторов» → ppt-master + Qwen-креатив
- «Ответь на возражение клиента» → DeepSeek-логика + Qwen-убеждение

**Реализация:**
1. Профиль: `~/.hermes/profiles/salesman/`
2. Telegram-бот: `@HermesSalesBot` (или личный чат)
3. Модели: DeepSeek V3 (primary) + Qwen-Max (creative fallback)
4. Скиллы: все business + презентации + аналитика
5. `cross-model-review` для важных КП (две модели → консенсус)
6. Эскалация сомнительных сделок → Море

**Конфигурация:**
```yaml
model:
  base_url: https://api.deepseek.com/v1
  default: deepseek-chat
  provider: custom
auxiliary:
  creative:
    provider: qwen-max
    model: qwen-max
    base_url: https://dashscope-intl.aliyuncs.com/compatible-mode/v1
    api_key_env: DASHSCOPE_API_KEY
agent:
  disabled_toolsets: [delegation]
  channel_prompt: |
    Ты продажник Hermes — AI-агент по продажам.
    Работаешь с: металлами, трубами, нефтегазом, AI-услугами.
    Используй Qwen для креатива, DeepSeek для логики.
    Сомнительные сделки → эскалировать Море (@Poliakarm).
```

**Метрики продажника:**
- КП отправлено / принято
- Среднее время ответа (< 5 мин)
- Конверсия из запроса в КП
- % эскалаций к Море

**Затраты:**
- DeepSeek V3: ~$10-20/мес
- Qwen-Max: ~$15-30/мес
- Итого: ~$30-50/мес на продажника
