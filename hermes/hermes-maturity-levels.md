# 15 уровней Hermes Agent: от чатбота до автономной системы 24/7

> Полный конспект статьи. Оригинал — telegraph.

---

## Фаза 1: Основа (уровни 1–3)

### Уровень 1. Одноразовые запросы

Агент делает действия в среде: читает файлы, ищет в интернете, запускает команды, правит код.

**Главная ошибка:** формулировать вопросы так, будто Hermes только пишет текст. Хорошая задача = действие + проверяемый артефакт.

### Уровень 2. Память и SOUL.md

Долговременные факты о пользователе, проектах, стиле. SOUL.md задаёт личность и рабочие правила. Один и тот же вопрос получает разный ответ у разных пользователей.

**Как:** `~/.hermes/SOUL.md`, 50-80 строк: голос, область работы, правила, запреты.

### Уровень 3. Slash commands

`/background`, `/steer`, `/queue`, `/model`, `/reasoning` — параллельная работа внутри одной сессии.

**Главная ошибка:** ждать завершения каждой задачи. `/background` удваивает пропускную способность.

## Фаза 2: Усиление (уровни 4–7)

### Уровень 4. Skills и правильная модель под задачу

Skills — загружаемые по требованию инструкции. Превращают общего агента в специалиста.

**Экономика моделей:**
- Web research → дешёвая модель
- Финальный синтез → сильная модель
- Code review → модель с сильным reasoning
- Массовые подагенты → бюджетная модель
- Локальные рутинные → self-hosted

### Уровень 5. MCP: подключение внешнего мира

GitHub, Notion, Slack, Gmail, Calendar, базы данных, API.

**Главная ошибка:** подключить 15 MCP сразу. Каждый сервер добавляет схемы в контекстное окно → дороже и шумнее.

### Уровень 6. Подагенты и параллельное выполнение

`delegate_task` запускает изолированных подагентов. Параллельные рабочие ветки: один исследует, второй кодит, третий проверяет.

**Важно:** delegation имеет накладные расходы. Простую задачу — главным агентом.

### Уровень 7. Асинхронная работа: goals, cron, checkpoints

- **Goals:** `/goal` + judge-модель решает, достигнут результат или продолжать
- **Cron jobs:** задачи по расписанию в изолированных сессиях → доставка в Telegram/DC/Slack
- **Checkpoints:** снапшоты рабочей директории, `/rollback` при ошибке

**Главная ошибка:** запускать autonomy без границ, rollback и бюджета.

## Фаза 3: Автономность (уровни 8–15)

### Уровень 8. Multi-profile architecture

Изолированные профили: Scout, Analyst, Briefer, Coder. У каждого свой SOUL.md, config, память, навыки, cron.

**Главная ошибка:** копировать один SOUL.md во все профили. Смысл — изоляция роли.

### Уровень 9. Самоулучшающаяся база знаний

Wiki-подход: interlinked markdown-база (Obsidian). Знания за пределами лимита памяти.

### Уровень 10. Kanban orchestration

SQLite-доска задач с зависимостями. Dispatcher подхватывает готовые. Для 10+ задач с зависимостями.

### Уровень 11. Voice mode

STT/TTS через платформы. Голосовое сообщение → расшифровка → задача → аудиоответ.

### Уровень 12. Browser automation

Браузер для сайтов без API. Динамические страницы, формы, личные кабинеты.

**Главная ошибка:** использовать браузер там, где есть API. Медленнее, хрупче, дороже.

### Уровень 13. API server

Hermes как OpenAI-compatible HTTP endpoint (`:8642/v1/chat/completions`). Интеграции с таблицами, кастомные UI.

### Уровень 14. IDE integration (ACP)

Hermes как ACP-сервер в VS Code, Zed, JetBrains. Чат, diff, терминал прямо в редакторе.

### Уровень 15. Profile distributions

Упаковка профиля как git-репозитория. Установка одной командой: `hermes profile install github.com/user/agent`.

## Экономика токенов

- **Правильная модель:** дёшево для классификации/routing, дорого для синтеза
- **wakeAgent gates:** скрипт проверяет изменения, LLM только если нужно
- **no_agent mode:** uptime/disk/file watchers без LLM
- **Pre-run scripts:** данные собираются детерминированно
- **Lean toolsets:** только нужные инструменты
- **Compression:** `threshold: 0.40`
- **Budget caps:** `daily_max_usd`, `session_max_usd`, `monthly_max_usd`

## Где останавливаются

- **Большинство:** уровни 1-2 (умный ассистент)
- **Основное усиление:** 3-7 (фон, навыки, cron)
- **Система:** 8-10 (профили, wiki, kanban)
- **15 не обязателен:** соло-основателю хватит 7-10

**Практичный маршрут:** уровень, снимающий текущую боль → настройка → проверка на workflow → дальше когда мало.
