# OpenRath — Session как first-class объект

**Источник:** Hugging Face / статья, 23.06.2026

## Суть

OpenRath вводит объект Session первого класса для мультиагентных систем. Fork, merge и replay — явные операции прямо в рантайме.

PyTorch-подобная модель: история вызовов инструментов, ветвление и память живут внутри единого объекта Session, а не восстанавливаются из внешних логов.

## Ключевые фичи

- **Fork** — ветвление сессии для параллельных гипотез
- **Merge** — слияние веток обратно
- **Replay** — воспроизведение сессии с той же историей вызовов
- **Инспекция** — трассировка из коробки

## Применимость к Hermes

- У нас: Obsidian vault + `session_search` (1690 сессий, без ветвления)
- У них: рантайм-объект с явным fork/merge
- Потенциал: для multi-agent сценариев (delegate_task с ветвлением) — можно взять как reference
