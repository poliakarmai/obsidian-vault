## 🎯 Что релевантно для Hermes

### 1. Harness > Model — мы это уже знаем
> «Качество агента зависит больше от харнеса, чем от модели.»

GPT-4 с хорошим харнесом обошёл GPT-5 на бенчмарках. **У нас:** конституция, скиллы, memory-preload, communication-protocol — это и есть наш харнес. DeepSeek v4 не самая мощная модель, но обвязка вытягивает.

### 2. Мультиагентность = контроль контекста
> «Это ничто более, чем способ контролировать контекст.»

**У нас:** delegate_task, саб-агенты для сложных задач (аудит 3 эшелона, анализ транскриптов). Правильно делаем — разгружаем корневой контекст.

### 3. Pipeline planner→implementer→reviewer
Устоявшийся паттерн, работает всегда. **У нас:** spec-driven-development, subagent-driven-development, adversarial-review. Совпадает.

### 4. MCP — инструменты, не «куски агента»
Anthropic переделали MCP: теперь это просто набор инструментов, а не «агент as a service». **У нас:** MCP-серверы (bybit-ws, dexscreener, osint-tools, tavily) — правильно, что инструменты, а не агенты.

### 5. Скиллы решают проблему контекста
Загружаются по требованию, не висят в промпте постоянно. **У нас:** именно так работает `skill_view()` — 50+ скиллов, грузятся только нужные. Минус (редеплой при обновлении) — у нас решается через `skill_manage(action='patch')`.

### 6. Санитайзинг всех входов
Prompt injection через HTML, MCP, результаты тулов. **У нас:** `input-guardrails` skill, `safety-guard` для деструктивных операций. Надо усилить — добавить санитайзинг результатов тулов.

### 7. Не более 7-8 агентов для таргетированной системы
**У нас:** Море + 2 тенанта + саб-агенты по задачам. В пределах нормы.

### 8. Middleware решает всё
OpenTelemetry, трейсинг, метрики — через middleware, не дублируя код. **У нас:** пока нет единого middleware. Надо подумать.

### 9. Прототипируй в n8n, фиксируй в коде
**У нас:** нет n8n, но есть cronjob + delegate_task для автоматизации.

### 10. A2A — пока не работает
У Google три сервера с A2A несовместимы, клиентов нет. **Вывод:** не тратить время на A2A, ждать.
