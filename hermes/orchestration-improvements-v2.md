## 5 предложений

### 1. Обновиться до upstream и перейти на platforms-as-plugins

**Проблема:** наш патч gateway/run.py хрупкий — 425 коммитов разницы, при обновлении почти гарантированно сломается.  
**Решение:** дождаться когда `migrate/platforms-to-plugins` вольётся в main (уже активная разработка, 488 файлов), обновиться, убрать патч.  
**Эффект:** Telegram-платформа станет плагином с нативной поддержкой профилей. channel_profiles → конфиг плагина. Меньше хаков, чище архитектура.

**Срок:** 1-2 недели (ждать мержа в main)  
**Риски:** патч может не пережить обновление — нужен бэкап gateway/run.py и план отката

---

### 2. Включить approve/deny gate для memory/skill writes

**Проблема:** пользователи могут случайно или намеренно изменить системную память/скиллы. Нет защиты от дурака.  
**Решение:** новый механизм из upstream — все записи в memory и скиллы проходят через гейт. Админ (ты) подтверждает или отклоняет.  
**Эффект:** безопасность без потери гибкости. Пользователи предлагают улучшения, ты контролируешь.

**Срок:** сразу после обновления Hermes  
**Риски:** минимальные — фича уже в main

---

### 3. Авто-онбординг: скрипт `hermes-tenant onboard @username`

**Проблема:** текущий процесс — ручной: создать профиль, .env, age-шифрование, channel_profiles, Obsidian vault. 15 минут на пользователя.  
**Решение:** скрипт который одной командой:
1. Создаёт `/home/hermes-{username}/`
2. Инициализирует профиль `~/.hermes/profiles/{username}/`
3. Генерит `.env.enc` с age-ключом
4. Добавляет запись в `channel_profiles`
5. Создаёт Obsidian vault
6. Отправляет приветственное сообщение

**Эффект:** подключение пользователя за 30 секунд вместо 15 минут.

**Срок:** 1-2 дня разработки  
**Риски:** низкие — изолированная операция

---

### 4. Per-user cost tracking + квоты

**Проблема:** все пользователи используют одни и те же API-ключи, нет понимания кто сколько жрёт токенов.  
**Решение:** расширить наш `cost_tracker` на per-user метрики:
- Считать токены на пользователя из логов Hermes
- Дневной лимит (например 50K токенов для обычных, unlimited для poliakarm)
- Алерт при приближении к лимиту
- Еженедельный отчёт: кто сколько потратил

**Эффект:** контроль расходов, база для монетизации, защита от перерасхода.

**Срок:** 2-3 дня  
**Риски:** нужен парсинг логов Hermes на per-session уровне

---

### 5. Per-tenant Obsidian vault + изоляция

**Проблема:** сейчас все пользователи разделяют одну память Hermes (Obsidian vault). Илья видит trading-заметки, ты видишь чужие данные.  
**Решение:** каждому пользователю свой vault:
- `/home/hermes-{user}/obsidian-vault/` с 00-Index, memory, user-profile
- В конфиге профиля: `OBSIDIAN_VAULT_PATH=/home/hermes-{user}/obsidian-vault/`
- При старте сессии агент загружает ТОЛЬКО свой vault
- Общие знания → shared vault read-only

**Эффект:** полная изоляция данных. Пользователи не видят чужие заметки. Безопасность и приватность.

**Срок:** 1 день (уже частично реализовано для Ильи)  
**Риски:** нужно проверить что shared-skills не пересекаются с vault
