# @Apolaibot — Командное внедрение AI

Гайд для тимлидов и руководителей по внедрению AI-агентов в команду.

## 1. Оценка готовности

Перед внедрением ответьте на 3 вопроса:
- Есть ли в команде повторяющиеся рутинные задачи? (код, документы, письма)
- Есть ли автоматическая проверка результата? (тесты, ревью, линтер)
- Готов ли кто-то быть «AI-чемпионом» — отвечать за внедрение?

Если все три ответа «да» — команда готова.

## 2. Выбор первых задач

**Начинайте с малого.** Первые 2 недели — только задачи где:
- Результат легко проверить (тесты проходят / документ открывается)
- Ошибка агента не ломает продакшен (изолированная среда)
- Задача занимает у человека >30 минут (окупает время настройки)

**Хорошие первые задачи:**
- Генерация тестов
- Форматирование документов
- Написание release notes
- Рефакторинг в отдельной ветке

**Плохие первые задачи:**
- Правка продакшен-кода напрямую
- Задачи без чётких критериев «готово»
- Всё что требует доступа к секретам/платежам

## 3. Настройка рабочего окружения

```bash
# 1. Создать изолированное рабочее пространство
python3 worktree_mgr.py create ~/project --agent team-agent

# 2. Настроить STATUS.md для отслеживания
python3 status_md.py init "Рефакторинг модуля X" --steps 5

# 3. Запустить конституционную проверку
python3 const_check.py --file plan.md

# 4. После завершения — слить изменения
python3 worktree_mgr.py merge ~/project --agent team-agent --message "AI: рефакторинг модуля X"
```

## 4. Правила командной работы

### Для разработчиков
- **Формулируйте задачу как issue.** Не «почини баг», а «исправь NullReferenceException в UserService.Create() — тест test_create_user_fails»
- **Проверяйте результат.** Агент — не замена код-ревью. Каждый PR от агента проходит обычный процесс.
- **Учите агента.** Если агент ошибся — поправьте в том же чате. Он запомнит контекст.

### Для тимлида
- **Назначьте AI-чемпиона.** Один человек в команде отвечает за промпты, настройку, метрики.
- **Ведите метрики.** Сколько задач решено агентом, сколько сэкономлено часов.
- **Стандартизируйте.** Создайте шаблоны промптов для типовых задач команды.

### Для руководителя
- **ROI считайте просто.** Часы ручной работы × ставка разработчика vs стоимость подписки.
- **Начинайте с одного агента.** Не раздавайте доступ всем сразу. Пусть AI-чемпион обкатает.
- **Собирайте фидбек.** Раз в 2 недели — опрос команды: что работает, что нет.

## 5. Шаблоны промптов

### Для код-ревью
```
Проверь этот PR:
- Соответствие код-стилю проекта
- Потенциальные баги и крайние случаи
- Производительность (N+1 запросы, утечки памяти)
- Безопасность (инъекции, утечка данных)
Результат: список проблем с указанием файла и строки.
```

### Для рефакторинга
```
Отрефакторь модуль {module}:
- Выдели повторяющийся код в утилиты
- Улучши читаемость (имена переменных, декомпозиция)
- Добавь type hints
- Не меняй публичное API
Результат: отдельная ветка с коммитом.
```

### Для документации
```
Опиши API эндпоинт {endpoint}:
- Назначение и контекст использования
- Входные параметры с примерами
- Выходной формат с примером ответа
- Коды ошибок и их значение
- Пример curl-запроса
```

## 6. Антипаттерны

| ❌ Неправильно | ✅ Правильно |
|---------------|-------------|
| «Сделай красиво» | «Выровняй отступы, переименуй x → user_count, добавь docstring» |
| Агент коммитит в main | Агент работает в worktree → PR → ревью → merge |
| 10 разработчиков, 1 агент | 1 агент на команду, очередь задач через agent_queue.py |
| «AI всё сломаal» | «AI сделал 3/5 задач правильно, остальные поправили за час» |
