## 1. Навыки (Skills) + MCPHub

### Концепт
- «Фабрика навыков» — набор инструментов, ИИ сам выбирает и вызывает нужный
- Навык = программа ИЛИ промпт
- Добавлять может: пользователь, организация, админ (on-prem)
- Искра сама понимает какой навык вызвать под задачу

### MCPHub
- Безопасная песочница для запуска навыков-программ
- MCP-серверы, API-интеграции, специализированные инструменты
- Изоляция: навык не может навредить данным или системе
- «Расширяемость без риска»

### Что у нас уже есть
- Hermes skills + using-hermes-skills (роутер)
- Нет: user-customizable skills через UI
- Нет: безопасной песочницы уровня MCPHub (у нас terminal с safety-guard, но не полноценная изоляция)

### Что можно взять
- UI для добавления навыков пользователем (не только админом через skill_manage)
- Авто-определение нужного навыка (улучшить router)
- Изоляция через контейнеры/podman для пользовательских навыков

## 7. Агентская файловая система (AI-driven FS)

### Концепт
Не просто «агент умеет создавать файлы», а полноценная файловая система для AI-агентов:
- **Строгие версии файлов** — каждая версия восстановима
- **Optimistic concurrency** — два агента не перетирают работу друг друга
- **Lock на путь** — атомарные операции над файлами
- **Артефакты привязаны к сообщениям** — удаление/изменение аудируемо
- **Runbook для rollout/rollback** — формальная процедура отката
- **Консистентность БД + бэкапы** — после аварии поднимается рабочее состояние, а не «примерно то же»

Ключевая мысль: красивая демка ≠ готовность к корпоративному контуру. Без этой «скучной» инфры AI-платформа опасна для реального бизнеса.

### Что у нас уже есть
- session-based изоляция через профили Hermes
- hermes-backup (бэкап перед опасными операциями)
- safety-guard (перехват rm -rf)
- Нет: версионирования файлов
- Нет: конкурентного доступа агентов к файлам
- Нет: привязки артефактов к сообщениям
- Нет: формального runbook для rollback

### Что можно взять
- Файловая система с версионированием как слой над Obsidian vault
- Lock-менеджер для параллельных агентов
- Привязка каждого созданного файла к ID сообщения (audit trail)
- Rollback до любой версии файла по команде

## 8. Смена модели: Qwen 3.5 → Qwen 3.7

### Концепт
Апгрейд базовой модели как мотора платформы:

- **Надёжнее в агентных сценариях** — держит много инструментов в одном прогоне, длинные цепочки без развала
- **Заточена под офис/продуктивность** — тексты, таблицы, документы, разборы встреч
- Бесшовно: обновление уже применено, включать ничего не нужно

### Что для нас
- Прямой аналог: выбор модели для @Apolaibot (сейчас deepseek-chat/v4-pro)
- Важно: модель должна быть заточена под агентные сценарии (tool use, long context, multi-step)

## 11. Организации и биллинг (июнь)

### Концепт
- **Рабочие пространства для организаций** — создать организацию может любой пользователь, сразу со Старта
- **Оплата за сотрудников** — админ оплачивает использование для своей команды
- **Авто-переключение контекста** — после создания организации сразу попадаешь в её рабочий контекст
- **Честный биллинг** — при переходе на более высокий тариф лимиты сбрасываются сразу (не ждать след. месяца)
- **Стабильность** — длинные ответы не обрываются таймаутом, переключение чатов не убивает стрим, текст в поле ввода не пропадает пока модель печатает

### Что у нас уже есть
- multi-tenant изоляция (hermes-orchestrator, профили)
- Нет: биллинга и тарифов
- Нет: paid organizations с админ-панелью
- Нет: сброса лимитов при апгрейде

### Что можно взять
- Организации как надстройка над multi-tenant: админ → сотрудники → биллинг
- Авто-переключение в контекст организации при создании
- Инстантный сброс лимитов при апгрейде тарифа

## 17. Общий чат + Очередь агентов + Стриминг + Оценки (июнь)

### Концепт
- **Общий чат в пространствах** — @упоминания с подсветкой, уведомления, пересылка сообщений и файлов из личных чатов. @Искра отвечает в общем чате.
- **Очередь агентов** — нет отказа «агенты заняты». Задача ждёт, агент освободился → подхватывает автоматически.
- **Непрерываемый стриминг** — обновил страницу или переключил чат → ответ догонит. Стрим переподключается, сообщения не теряются.
- **Оценки 👍👎** — под каждым ответом AI. Фидбек попадает в контекст, AI учитывает в следующих ответах. Сообщения можно удалять.
- **Приглашения по ссылке** — админ создаёт ссылку-приглашение в организацию → сотрудник переходит → сразу в нужных группах.

### Что у нас уже есть
- Нет: shared-чатов между тенантами
- Нет: очереди задач (сейчас delegate_task параллельно, но без очереди)
- Нет: реконнекта стрима при обрыве
- Нет: фидбек-лупа 👍👎 → контекст

### Что можно взять
- Shared-чаты для команд в @Apolaibot
- Очередь задач: вместо отказа → «ваш запрос в очереди, агент ответит через ~X сек»
- Реконнект стрима (актуально для веб-клиента)
- Фидбек-петля: оценка → улучшение ответов
