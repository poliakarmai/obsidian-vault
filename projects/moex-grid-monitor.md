# MOEX Bollinger Grid Монитор

**Статус:** 🟡 Идея / Будущий проект  
**Дата:** 2026-06-19  
**Связано:** [[hermes/bybit-ws-full]]

## Концепция

Перенести Bollinger Grid монитор с крипты на рынок акций РФ (MOEX).  
Ядро переиспользуется, меняется только API-слой.

## Что уже есть (bybit-ws → переиспользовать)

- `ws_client.py` — BB-кеш (замена на REST MOEX ISS)
- `rpc.py` — `/rpc/scan` эндпоинт, скоринг
- `gridsignal-bot.py` — Telegram-бот @Gridbolbot
- Вся архитектура RPC + крон + MCP

## Что нужно сделать

| Компонент | Сейчас | Будет |
|-----------|--------|-------|
| API | Bybit v5 (HMAC) | MOEX ISS (открытый REST) |
| Торговля | Фьючерсы 24/7, плечи | T+2, сессия 10:00–18:45 |
| Данные | WS-поток | REST-свечи ISS history |
| Исполнение | Bybit API | Брокерский API (Тинькофф/Альфа) |
| Скоринг | BB + RSI + оборот | BB + RSI + объём MOEX |

## Оценка трудозатрат

- Монитор без автотрейдинга: **3-4 дня**
- С автоисполнением: **1-2 недели** (зависит от брокерского API)

## Главная засада

Брокерский API: Тинькофф Инвестиции дают REST + gRPC, Альфа — свой.  
Потребуется отдельный слой абстракции для ордеров.

## Приоритет

Низкий. Отложено до освобождения от текущих задач по bybit-ws и Hermes.
